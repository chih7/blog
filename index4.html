ubuntu突然出现打开系统设置慢 sudo时出现无法解析主机的解决办法

2014-04-06

突然间发现我的ubuntu12.04打开系统设置变得非常慢，正不知如何解决的时候。发现sudo时出现“无法解析的主机 XXXX" 的提示。马上就联想到不久前改过hosts，原来是主机名无法解析了，这样问题就好解决了

$ sudo vim /etc/hosts

加入以下行

127.0.0.1       （你现在的主机名）
127.0.1.1       （你现在的主机名）

之后 打开系统设置就非常快了

virtualbox主机访问虚拟机

2014-03-02

在virtualbox中，如果选择默认的网卡设置，也就是网络地址转换(NAT)，是不能让宿主机访问到虚拟机的，如果想让两者相互连通，可以在保留默认的NAT网卡的基础上增加一张Host-Only的网卡。

需要注意的是，如果你的宿主机是Linux的话，安装virtualbox后是没有一张Host-Only的网卡的，你需要在virtualbox的设置里添加这张网卡。

VMware中使easy install失效

2012-09-05

用Typcial或custom方式安装虚拟机时，会出现Easy Install，使得安装过程自动进行，但这种自动的安装会有一些问题，比如装Linux时的分区，ubuntu的中文支持也会有问题。 更多的，对于像我这种学生来说，虚拟机有很大的作用是用来学习系统的安装的，这时Easy Install就显得多余了，那我们要怎么取消Easy Install呢？下面是一种不错的方法。

  1. 用Typcial或custom方式进入虚拟机的安装。
  2. 出现Easy Install Information时，可以随便填写，这个在后来的实际安装过程中是用不着的。
  3. 按你的需要配置虚拟机
  4. 在Ready to Create Virtual Machine页中，需要把Power on this virtual machine after creation前的勾去掉，Finish
  5. 接下来配置你刚刚创建的虚拟机。
  6. 把使用autoinst.iso的CD/DVD这个硬件直接remove或选为Auto detect，同样更改Floppy这个硬件，OK
  7. 现在启动虚拟机安装系统时就不会进入Easy Install了。
